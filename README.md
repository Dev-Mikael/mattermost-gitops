# mattermost-gitops

GitOps repository for deploying **Mattermost** on Kubernetes (on-prem / local)
using **FluxCD**, **Sealed Secrets**, **MetalLB**, **NGINX Ingress**, and **cert-manager**.

---

## Quick Start

```bash
# 1. Copy and fill in your values
cp .env.example .env
nano .env

# 2. (On-prem only) Set up the server first
sudo bash scripts/02b-setup-kubeadm.sh

# 3. Run the full automated bootstrap
bash bootstrap.sh
```

For the full step-by-step guide see **Mattermost_Deployment_Guide.docx**.

## Day-2 Operations

| Task | Action |
|------|--------|
| Upgrade Mattermost | Edit `version:` in `apps/mattermost/mattermost.yaml`, commit & push |
| Scale replicas | Edit `replicas:` in `mattermost.yaml`, commit & push |
| Rotate DB password | Re-run `scripts/04-wait-and-seal.sh` with new `DB_PASSWORD` in `.env` |
| Force Flux sync | `flux reconcile kustomization flux-system --with-source` |
| View all Flux status | `flux get all -A` |
| View Mattermost logs | `kubectl logs -n mattermost -l app=mattermost -f` |

## DNS Setup (modumichael.me)

Create an **A record** at your DNS provider:

| Host | Type | Value |
|------|------|-------|
| `@` or `modumichael.me` | A | `<your server public IP>` |

TLS certificates are issued automatically by Let's Encrypt via cert-manager.
